### Task 4: Fix `extractExplicitCurrency` regex (too loose)

**Files:**
- Modify: `src/queue.ts`

**Current code (line 357):**
```ts
function extractExplicitCurrency(text: string): string | null {
  const match = text.toUpperCase().match(/\b[A-Z]{3}\b/);
  return match?.[0] ?? null;
}
```

This matches any 3-letter word: "EAT", "BUS", "THE", "FOR", "DAY", etc. It should only match known ISO 4217 currency codes.

This function is only used in the regex fallback path (when `OPENAI_API_KEY` is absent), so this fix prevents bad currency data from entering the DB in degraded mode.

**Step 1: Replace the regex with an allowlist of common currencies**

The goal is not to enumerate every ISO 4217 code, but to cover currencies the app realistically encounters. Use a Set of the most common codes:

```ts
const KNOWN_CURRENCIES = new Set([
  "USD", "EUR", "GBP", "JPY", "AUD", "CAD", "CHF", "CNY", "HKD", "SGD",
  "PHP", "INR", "MYR", "THB", "IDR", "VND", "KRW", "TWD", "NZD", "SEK",
  "NOK", "DKK", "MXN", "BRL", "ARS", "ZAR", "AED", "SAR", "TRY", "RUB"
]);

function extractExplicitCurrency(text: string): string | null {
  const words = text.toUpperCase().match(/\b[A-Z]{3}\b/g) ?? [];
  return words.find(w => KNOWN_CURRENCIES.has(w)) ?? null;
}
```

Place `KNOWN_CURRENCIES` as a module-level constant just above the `extractExplicitCurrency` function.

**Step 2: Run type check**


Expected: no errors.

**Step 3: Run tests**

Expected: all existing tests pass (this function is in the no-OpenAI fallback path, which is tested in queue.test.ts).

**Step 4: Commit**

### Task 7: Lock CORS origin to configurable env var

**Files:**
- Modify: `src/app.ts`
- Modify: `src/types.ts`
- Modify: `wrangler.toml`

Current CORS config:
```ts
cors({ origin: "*", ... })
```

This allows any origin to call the API with a valid Telegram initData token. It should be restricted to the Cloudflare Pages domain hosting the Mini App.

**Step 1: Add `CORS_ORIGIN` to `src/types.ts`**

In the `Env` interface, add:
```ts
CORS_ORIGIN?: string;
```

Making it optional means the app still works if the var is not set (falls back to `"*"` for local dev).

**Step 2: Update CORS config in `src/app.ts`**

Replace:
```ts
cors({
  origin: "*",
  ...
})
```

With:
```ts
cors({
  origin: (origin, c) => c.env.CORS_ORIGIN ?? "*",
  ...
})
```

This uses the Hono CORS `origin` function signature — it receives the request origin and returns the allowed origin string. In production, only requests from `CORS_ORIGIN` will get CORS headers.

**Step 3: Add `CORS_ORIGIN` to `wrangler.toml` under `[vars]`**

